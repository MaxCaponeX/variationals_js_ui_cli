# Variational Bot

Автоматический торговый бот для платформы [omni.variational.io](https://omni.variational.io).
Поддерживает два режима торговли: **Single** (одиночные позиции) и **Delta Neutral** (парные противоположные позиции).
Управляется через графический интерфейс (GUI).

## Требования

Перед запуском нужно установить **Node.js**.

Node.js — это среда выполнения JavaScript. npm — менеджер пакетов, идёт вместе с Node.js.

**Windows / macOS / Linux:**
Перейди на [https://nodejs.org](https://nodejs.org) и скачай версию **LTS** (рекомендуемую).
Установи как обычную программу.

Проверить установку (в терминале):

```bash
node --version   # должно показать v18.x.x или выше
npm --version    # должно показать 9.x.x или выше
```

---

## Установка с нуля

1. Скачай архив с проектом (либо клоинруйте через git clone) и распакуй в любую папку
2. Открой терминал (Terminal на macOS/Linux, PowerShell на Windows) в папке проекта
3. Выполни:

```bash
npm install
```

Это установит все зависимости и автоматически создаст нужные папки и файл `config.json`.

После `npm install` в папке проекта автоматически появятся:

```
user_data/
  privatekeys/privatekeys.txt   ← сюда вставляй приватные ключи
  proxies/proxies.txt           ← сюда вставляй прокси
databases/                      ← база данных (создаётся при запуске)
config.json                     ← конфиг (создаётся автоматически)
```

## Первоначальная настройка

### Приватные ключи

Есть два способа добавить ключи:

**Через GUI** (таб **Аккаунты**):

1. Открой таб **Аккаунты**
2. В поле "Приватные ключи" вставь ключи — по одному на строку
3. Нажми **Сохранить**

**Вручную** — открой файл `user_data/privatekeys/privatekeys.txt` и впиши:

```
# Без метки — просто ключ:
0xYOUR_PRIVATE_KEY_1

# С меткой (отображается в логах):
MyWallet:0xYOUR_PRIVATE_KEY_2

# Строки начинающиеся с # — комментарии, игнорируются
```

> ⚠️ Никогда не отправляй этот файл никому и не коммить в Git.
> Файл уже добавлен в `.gitignore` и не попадёт в репозиторий.

---

### Прокси

Прокси — опционально. Если не используешь, бот запустится без них.

**Через GUI** (таб **Аккаунты**) или вручную в `user_data/proxies/proxies.txt`:

````
# Формат: http://логин:пароль@ip:порт
http://user:pass@1.2.3.4:8080
http://user:pass@5.6.7.8:3128

Прокси привязываются к кошелькам по порядку: первый прокси → первый кошелёк, второй → второй и т.д.
Если прокси меньше чем кошельков — оставшиеся работают без прокси.

---

### Создание базы данных

Перед запуском торговли нужно создать базу данных.

1. На главной вкладке нажми **Создать БД**
2. Введи пароль — он используется для шифрования приватных ключей
   (запомни его, он будет нужен при каждом запуске)
3. Выбери режим: **Single** или **Delta Neutral**
4. Нажми **Создать**

> Пароль по умолчанию если хочешь не вводить каждый раз: `capONE`

## Настройка параметров

Все параметры сохраняются в `config.json` через таб **Настройки** в GUI.

---

### Токены

Таб **Токены**. Здесь задаёшь с какими монетами работает бот.

| Параметр | Описание |
|----------|----------|
| **Ценовой диапазон** | Бот откроет сделку только если цена в этом диапазоне. Например, BTC 80000–110000 |
| **Плечо** | Диапазон кредитного плеча [мин, макс]. Рандомно берётся из диапазона |
| **Макс. спред %** | Максимально допустимый спред между bid и ask. Если спред выше — ждёт |
| **Смещение цены $** | Смещение лимитного ордера от рыночной цены в $ [мин, макс] |

Чтобы добавить новый токен — введи его тикер (например `AVAX`) и нажми **+**.
Чтобы удалить — нажми **×** рядом с токеном.

---

### Single Mode

Таб **Настройки → Single Mode**. Одиночная торговля на каждом кошельке независимо.

| Параметр | Описание |
|----------|----------|
| **Кол-во трейдов** | Сколько сделок делать за один цикл [мин, макс] |
| **Размер сделки $** | Фиксированный размер в USDC [мин, макс]. Если [0,0] — используется % |
| **Размер сделки %** | Процент от баланса [мин, макс] |
| **Закрыть всё перед стартом** | Закрывать открытые позиции перед новым циклом |
| **Типы ордеров — открытие** | `limit` и/или `market` — бот случайно выбирает из включённых |
| **Типы ордеров — закрытие** | То же самое для закрытия |
| **Стоп-лосс** | Включить/выключить. Отступ в % или $ |
| **Отмена ордеров при закрытии** | Отменять незаполненные pending ордера |
| **Закрывать позиции** | Закрывать открытые позиции при вызове "Закрыть всё" |

---

### Delta Neutral

Таб **Настройки → Delta Neutral**. Группы кошельков открывают противоположные позиции (Long + Short) для нейтрализации рыночного риска и сбора объёма.

| Параметр | Описание |
|----------|----------|
| **Кошельков в группе** | Сколько кошельков в одной группе [мин, макс]. Обычно [2,2] |
| **Время удержания позиции** | Сколько секунд держать позиции перед закрытием [мин, макс] |
| **Разброс позиций** | Насколько позиции могут отличаться по размеру (для маскировки) [мин, макс] как % |
| **Тип ордера** | `limit` или `market` для открытия позиций группы |

---

### Общие настройки

Таб **Настройки → Общие**

| Параметр | Описание |
|----------|----------|
| **Потоки** | Сколько кошельков/групп работают одновременно |
| **Попытки при ошибке** | Сколько раз повторять при неудаче |
| **Перемешивать кошельки** | Случайный порядок обработки кошельков |
| **Между потоками (сек)** | Задержка перед стартом каждого следующего потока [мин, макс] |
| **Между ордерами (сек)** | Пауза между последовательными ордерами одного аккаунта |
| **Между открытием ордеров (delta)** | Пауза между открытием ордеров в группе |
| **Между закрытием ордеров (delta)** | Пауза между закрытием ордеров в группе и в режиме "Закрыть всё" |
| **После аккаунта (сек)** | Пауза после завершения аккаунта/группы |
| **После продажи всего (сек)** | Пауза после sellAll перед следующим действием |

### Telegram уведомления

Для получения уведомлений о сделках нужен Telegram бот.

**Шаг 1 — Создать бота:**
1. Открой [@BotFather](https://t.me/BotFather) в Telegram
2. Отправь `/newbot`
3. Придумай имя и username боту
4. Скопируй **токен** (выглядит как `1234567890:ABCdef...`)

**Шаг 2 — Узнать свой Telegram ID:**
1. Открой [@userinfobot](https://t.me/userinfobot)
2. Отправь `/start`
3. Скопируй число из поля `Id`

**Шаг 3 — Вставить в настройки:**
В GUI таб **Настройки → Общие** вставь токен и ID.
Или вручную в `config.json`:
```json
"telegram": {
  "botToken": "1234567890:ABCdefGHI...",
  "userIds": [123456789]
}
````

**Что приходит в Telegram:**

- 📂 Группа открыла позиции (с деталями каждого аккаунта)
- 📁 Группа закрыла позиции (с Profit / Volume / 100k$ cost)
- Итоговый отчёт после завершения всех аккаунтов

> Если не нужны уведомления — оставь поля пустыми, бот работает без Telegram.

## Режимы запуска

На главной вкладке есть 5 кнопок запуска:

| Режим                   | Описание                                                                    |
| ----------------------- | --------------------------------------------------------------------------- |
| **Single**              | Торгует на каждом кошельке по N трейдов. Ордера: limit/market по настройкам |
| **Delta Neutral**       | Запускает группы кошельков, открывает Long+Short, ждёт, закрывает           |
| **Закрыть все позиции** | Закрывает все открытые позиции на всех кошельках                            |
| **Статистика**          | Парсит и выводит текущее состояние по кошелькам                             |

При нажатии любой кнопки появится окно с запросом пароля БД.

## Стоп

Кнопка **Стоп** прерывает все операции немедленно:

- Текущие ожидания сна — прерываются
- Следующие потоки — не запускаются
- Уже выполняющиеся HTTP запросы — завершают текущий и останавливаются

После нажатия Стоп можно снова запустить без пересоздания БД.

## Частые вопросы

**Q: Забыл пароль от БД. Что делать?**
A: Пересоздай БД через кнопку **Создать БД** — старая перезапишется.

**Q: Бот пишет "No modules in database. Create database first."**
A: Нажми **Создать БД** перед запуском торговли.

**Q: Ошибка "No private keys found in file."**
A: Добавь приватные ключи в таб **Аккаунты** или в `user_data/privatekeys/privatekeys.txt`.

**Q: Прокси не работают / ERR_NO_SUPPORTED_PROXIES**
A: Проверь формат — должен быть `http://логин:пароль@ip:порт`.
Socks5 не поддерживается. Только HTTP прокси.

**Q: Можно запустить без GUI (CLI)?**
A: Да: `npm start` — запустит консольный интерфейс.
